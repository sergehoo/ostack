# Objectif du cycle global OStack

Examiner le projet actuel, identifier l’amélioration vérifiable de plus fort levier et produire un
plan cohérent qui applique chaque skill OStack installé.

Contraintes :

- préserver la rétrocompatibilité ;
- ne pas inventer de données ni de preuves ;
- signaler les actions bloquées ou non pertinentes ;
- associer chaque affirmation de réussite à une preuve exécutable ;
- ne proposer aucune mutation de production sans approbation humaine.
